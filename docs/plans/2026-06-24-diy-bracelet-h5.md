## 实现细节:

1. `app.js` 的商品数据与纯状态函数 -> 定义本地珠子商品数据、`RULES`、`filterProducts`、`addBraceletItem`、`removeBraceletItem`、`calculateSummary` 和 `calculateBeadPositions`，保证筛选、重复选珠、价格计算、最少 16 颗提示和椭圆轨道坐标均可脱离 DOM 测试。
2. `index.html` 的语义页面骨架 -> 提供标题栏、状态栏、手链舞台、功能操作、分类按钮、搜索框、商品列表和无结果提示；为商品、已选珠子和清空操作提供明确按钮标签与测试标识。
3. `styles.css` 的移动端视觉与响应式布局 -> 使用 CSS 变量统一颜色、间距、阴影和圆角；以 `100dvh`、安全区边距、底部抽屉和 375px 至 430px 断点实现设计稿布局；为空态、选中、焦点和按压状态提供清晰反馈。
4. `app.js` 的界面渲染和事件绑定 -> 使用 `textContent` 生成商品文案，按当前状态重绘商品网格、SVG 轨道上的珠子按钮和总价；实现分类/搜索、添加、删除和清空操作，并对本地异常商品与失效删除请求输出 `console.warn` 后安全返回。
5. `tests/app.test.mjs` -> 以纯函数覆盖商品筛选、增删、总价、数量状态和椭圆坐标；每个行为按 RED-GREEN-REFACTOR 垂直完成。
6. `README.md` -> 说明无需安装依赖的本地打开方式、支持的原型功能、测试命令和不包含的后端能力。

## 性能优化细节

- 商品和已选珠子仅有演示级小数据量；筛选或状态变化时才重绘相关区域，不使用动画循环。
- 商品图片使用 CSS 渐变生成演示珠子，不引入图片网络请求、预加载或大型图形依赖。
- 椭圆坐标在状态变化时一次计算，避免在滚动或指针移动中重复计算。

## 安全控制细节

- 搜索输入和商品字段一律以 `textContent` 写入 DOM，不解析为 HTML，避免 XSS。
- 页面不请求网络、不保存用户信息、不接第三方脚本或支付能力。
- 对缺失关键字段的本地商品数据跳过渲染并输出 `console.warn`，避免错误内容进入 UI。

## 边界处理细节

- 搜索或分类没有匹配商品时显示无结果空态；属于可接受输入，直接返回空数组。
- 同商品重复点击会生成独立的已选珠子并累计价格；不限制库存。
- 删除未找到的已选 ID 时输出 `console.warn` 并保持现有状态。
- 少于 16 颗时显示“珠子数量不足”，但保留继续添加和删除的能力。
- 空手链显示引导状态；清空后恢复该状态。
- 无效商品记录缺少名称、正价格或规格时不渲染，并输出 `console.warn`。
- 页面刷新不恢复已选状态；无持久化是已确认的原型范围。
- 小屏不横向溢出，按钮具有键盘焦点；不禁用浏览器缩放或原生滚动手势。

## 测试验证细节-测试驱动开发

测试方式：Node.js 单元测试 + 浏览器 E2E 冒烟测试。

E2E type: browser
Protocol strength: none
Reason: 本次只改变静态页面的可见渲染和 DOM 点击行为，没有后端、协议或跨服务边界；真实浏览器可验证最小用户旅程。

1. `filterProducts` -> 正常：按分类、名称、规格筛选；反向：无匹配关键词、未知分类、不同分类排除；边界：空关键词、纯空白、大小写差异；异常：缺失字段商品被忽略。
2. `addBraceletItem` / `removeBraceletItem` -> 正常：添加、重复添加、删除指定珠子；反向：删除不存在 ID、空列表删除、错误 ID 类型；边界：第一颗、16 颗阈值、清空后删除；异常：不完整商品不得加入。
3. `calculateSummary` / `calculateBeadPositions` -> 正常：正确总价、数量和坐标数量；反向：空选择总价为零、无效价格不计价、无效尺寸不排布；边界：0、1、16 和多颗珠子；异常：缺失数组或非法记录安全返回默认结果。
4. 浏览器 E2E 冒烟 -> 验证空态、分类筛选、搜索、点击商品后数量/总价变化、点击手链珠子删除和清空恢复空态；在 375px 与 430px 宽度检查页面无横向滚动和关键控件可见。

集成测试不适用：无接口、存储、网络或跨模块服务边界；纯函数单测与真实浏览器冒烟已覆盖该原型的行为风险。
